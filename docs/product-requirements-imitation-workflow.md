# 短片小说/故事内容生产工具需求文档

## 文档信息

- 文档版本：v1.1
- 更新时间：2026-06-20
- 适用项目：YouTube Analyse + InkOS 创作链路
- 产品方向：短片小说/故事内容生产工作台
- 文档目的：把当前项目仍然不足的地方整理成可执行需求，便于后续排期、开发和验收。

## 1. 产品结论

这个项目有潜力从“泛 YouTube 分析工具”收敛成“短片小说/故事内容生产工具”。更准确的主线应该是：

```text
YouTube 爆款故事视频
-> 提取字幕/原文
-> 清洗成可分析文案
-> 拆解故事结构、爽点、反转和节奏
-> 生成原创故事创作包
-> 调用 InkOS 或其他生成器产出草稿
-> 检查相似度、风险和质量
-> 多轮改写到可发布
```

当前项目已经完成了不少基础能力，但还需要继续补齐“生产闭环、质量闭环、规模化工作流、合规表达”四类能力。

## 2. 当前能力盘点

### 2.1 已具备能力

- YouTube 频道配置、视频同步和基础视频报告。
- 字幕/转录文本采集、翻译和报告证据展示。
- 视频报告页中的故事工坊：原始文本、清洗稿、清洗统计、清洗版本和短片小说结构分析。
- 仿写工厂：从报告生成 InkOS reference package，支持方向、输出类型、相似强度、目标长度等参数。
- InkOS 联动配置：支持通过命令调用 InkOS，并把结果保存回项目。
- 草稿保存、版本管理、状态更新和 Markdown 导出。
- 相似度检测：文本重合、重复短语、复用实体、结构/风格相似度、风险段落和修改建议。
- 质量门禁：根据检测结果判断 `pass`、`needs_revision`、`blocked`，并影响草稿状态。
- 降风险改写和多模式改写：节奏更快、开场更强、短剧对白、Shorts 口播等。
- 项目库：汇总创作项目，支持搜索、状态筛选、风险筛选、模板收藏和复用。
- 风格库：支持多报告风格融合。
- 首页工作流：展示创作流水线、候选选题和批量分析入口。

### 2.2 仍然不足

| 不足方向 | 当前表现 | 影响 | 优先级 |
| --- | --- | --- | --- |
| 产品命名与合规表达 | 页面仍大量使用“仿写/Imitation” | 容易形成洗稿心智，不利于长期产品化 | P0 |
| 主流程引导不够强 | 用户首次进入后不一定知道下一步做什么 | 新手转化低，功能像分散工具箱 | P0 |
| 原文清洗仍偏基础 | 清洗规则和人工校对体验还不够强 | 后续分析和生成质量不稳定 | P0 |
| 故事结构拆解还需更专业 | 爽点、悬念、反转、情绪曲线等还可继续结构化 | 生成稿容易只像“摘要扩写”，不像短片小说 | P0 |
| InkOS 生成结果可控性不足 | 参数、失败恢复、生成记录和成本提示还不完整 | 一键生成体验不稳定 | P1 |
| 相似度检测还偏规则化 | 缺少更细的语义、桥段和叙事节奏检测 | “像但不抄”的判断还不够可信 | P1 |
| 改写闭环还不够产品化 | 有改写动作，但缺少问题定位、改写前后对比和通过标准 | 用户不知道改到什么程度才算合格 | P1 |
| 项目库缺少生产排程 | 有项目筛选，但缺少排序、批量操作、状态看板 | 内容工作室难以规模化管理 | P1 |
| 选题与素材池不够系统 | 候选选题已有基础，但还缺少打分解释和批量决策 | 难以稳定筛出值得拆解的爆款样本 | P2 |
| 数据指标闭环不足 | 缺少完成率、通过率、改写次数等产品指标 | 后续难判断功能是否真的提升效率 | P2 |

## 3. 目标用户与核心场景

### 3.1 目标用户

- 短片小说/故事号创作者：想把爆款视频拆成可复用结构，再创作新故事。
- 内容工作室：需要批量研究竞品视频、沉淀模板、规模化产出文案。
- AI 创作工具用户：已经使用 InkOS、ChatGPT、Claude 等工具，但缺少稳定的上游拆解和质检链路。
- 跨语种内容团队：研究英文或其他语言的故事视频，转化成中文原创故事脚本。

### 3.2 核心场景

1. 用户输入或同步一条 YouTube 爆款故事视频。
2. 系统提取字幕，清洗成可读、可编辑、可保存的故事原文。
3. 系统拆解开场钩子、主角困境、信息差、爽点、反转、高潮和结尾悬念。
4. 用户选择新故事方向和输出类型，生成创作参考包。
5. 系统调用 InkOS 生成原创草稿，并保存版本。
6. 系统检测草稿与原文的相似度、复用实体和结构风险。
7. 用户根据检测结果一键改写，直到质量门禁通过。
8. 用户导出可发布文案，或沉淀为可复用结构模板。

## 4. 需求规划

### P0-1 产品命名与合规表达调整

**问题**

当前页面和字段中仍大量出现“仿写”“Imitation Factory”等表达。内部字段可以暂时保留，但面向用户的产品心智应转向“结构拆解、创作转化、原创改写、风险检测”。

**需求**

- 将用户侧主导航和页面文案从“仿写工厂”逐步改为“创作转化工坊”或“故事创作工坊”。
- 将“仿写项目”改为“创作项目”。
- 将“相似强度”表达为“结构保留强度”或“参考强度”。
- 保留“避抄规则、风险检测、质量门禁”等合规提醒。

**验收标准**

- 中文界面不再把“仿写”作为主要功能名称。
- 用户完成一次生成流程时，看到的是“参考结构并原创转化”的表达。
- 导出的 reference package 明确包含“不可复用内容”和“避抄边界”。

### P0-2 新手主流程引导

**问题**

功能已经较多，但新用户不一定知道应该从哪里开始，什么时候进入故事工坊，什么时候进入创作工坊。

**需求**

- 首页增加清晰的步骤流：同步视频、分析报告、清洗文案、拆解结构、生成草稿、风险检测、导出。
- 每个步骤显示当前数量和下一步动作。
- 报告页、故事工坊、创作工坊之间增加明确跳转。
- 对没有数据的页面提供可执行空状态，而不是只提示“暂无数据”。

**验收标准**

- 新用户能在首页看到下一步该做什么。
- 从一条已分析视频可以在 3 次点击内进入草稿生成。
- 空状态至少提供一个可执行按钮。

### P0-3 原文清洗与校对增强

**问题**

故事工坊已有清洗稿和版本，但清洗规则、人工校对体验和质量提示还可以增强。

**需求**

- 自动识别字幕残留、重复句、无意义口癖、异常断句和过短段落。
- 显示清洗前后差异，包括删除内容、合并段落和改写断句。
- 增加“清洗质量评分”和“需要人工确认”的提示。
- 支持一键恢复上一版清洗稿。

**验收标准**

- 每份报告都有原始字幕、清洗稿、清洗统计和版本记录。
- 用户能看到清洗过程移除了哪些噪声。
- 保存新清洗稿后，后续结构分析和创作参考包优先使用最新清洗稿。

### P0-4 短片小说结构拆解专业化

**问题**

目前结构分析已经具备基础字段，但还需要更贴近短片小说/故事号生产。

**需求**

- 增加结构字段：开场 5 秒钩子、前 30 秒留存机制、主角初始困境、身份差/信息差/能力差、第一爽点、反派阻力、公开反转、结尾悬念。
- 输出“可复用结构模板”和“不可复用内容清单”。
- 对每个结构节点提供原文证据片段。
- 支持用户手动编辑结构模板。

**验收标准**

- 每份故事分析至少包含 8 个短片小说专用结构字段。
- 创作工坊生成 reference package 时自动引用这些字段。
- 用户能区分“可复用机制”和“禁止复用设定/句子/人物”。

### P1-1 InkOS 调用与结果回流增强

**问题**

项目已经支持 InkOS 调用，但还需要更稳定的产品化体验。

**需求**

- 增加 InkOS 配置检测、失败原因提示和重试入口。
- 记录每次生成的参数、命令、reference package、输出结果和耗时。
- 支持生成前预览成本或风险提示。
- 支持选择历史 reference package 重新生成。

**验收标准**

- InkOS 未配置、命令失败、输出为空时均有明确提示。
- 每次生成都能追溯到所用参数和源报告。
- 用户可以对同一项目生成多个草稿版本。

### P1-2 相似度与风险检测升级

**问题**

当前检测已覆盖文本重合、重复短语、复用实体和风险段落，但仍偏规则化，需要更可信地判断“像但不抄”。

**需求**

- 增加语义相似度检测，识别换词但情节高度一致的段落。
- 增加桥段相似度检测，判断关键事件顺序是否过度重合。
- 增加叙事节奏检测，判断反转和高潮位置是否过度接近。
- 风险报告输出“必须修改”“建议修改”“可接受”三级。
- 质量门禁给出明确下一步动作：直接导出、继续改写、禁止发布。

**验收标准**

- 高风险草稿会被标记为 `blocked` 或 `needs_revision`。
- 风险段落能定位到具体片段和原因。
- 修改后重新检测，风险等级和质量门禁会更新。

### P1-3 改写闭环产品化

**问题**

已有降风险和多模式改写，但用户仍需要更直观看到“为什么改、改了哪里、是否通过”。

**需求**

- 改写前显示失败的质量检查项。
- 改写后显示前后对比、风险变化和通过情况。
- 支持针对单个风险段落局部改写。
- 支持改写目标：降低相似度、增强开场、提高节奏、短剧化、口播化、压缩篇幅。

**验收标准**

- 每次改写生成新草稿版本，不覆盖旧版本。
- 改写记录包含改写模式、原风险等级、新风险等级。
- 用户可以从风险报告直接发起局部改写。

### P1-4 项目库与生产管理增强

**问题**

项目库已经具备搜索、筛选和模板收藏，但还缺少更像内容生产台的管理能力。

**需求**

- 增加排序：最新创建、最新修改、风险最高、文本重合最高、草稿最多、已收藏模板。
- 增加批量操作：批量标记状态、批量导出、批量进入检测。
- 增加看板视图：参考包、待审查、待修改、可发布、已废弃。
- 增加“只看未通过质量门禁”的筛选。

**验收标准**

- 用户能快速找到需要继续修改的项目。
- 内容团队可以按状态推进项目。
- 高风险项目不会被误认为可发布。

### P2-1 批量选题与候选素材池

**问题**

首页已有候选选题和批量分析雏形，但还没有形成稳定素材池。

**需求**

- 对 recent videos 生成候选选题卡。
- 给出爆款潜力、故事适配度、结构复用价值和风险提示。
- 支持用户勾选多个候选视频进入分析队列。
- 支持按频道、题材、发布时间、观看表现排序。

**验收标准**

- 用户能从一批视频里筛出最值得拆解的前 N 个。
- 每个候选项都有可解释的推荐理由。
- 被选中的候选视频能批量进入分析任务。

### P2-2 风格与模板资产库

**问题**

风格融合和模板收藏已有基础，但还可以变成可持续复用的资产库。

**需求**

- 结构模板支持命名、标签、备注、适用题材和成功案例。
- 风格包支持来源报告列表、核心表达特征和禁用表达。
- 创作工坊可选择模板/风格包作为生成约束。
- 支持模板质量指标：被复用次数、通过率、平均风险等级。

**验收标准**

- 用户可以从项目库收藏结构模板。
- 模板能在新项目中直接复用。
- 模板列表能按题材和质量指标筛选。

### P2-3 产品指标与数据看板

**问题**

当前缺少衡量生产效率和质量提升的数据指标。

**需求**

- 增加工作台指标：本周分析视频数、生成项目数、草稿数、可发布数。
- 增加质量指标：质量门禁通过率、平均文本重合率、平均改写次数。
- 增加漏斗指标：已同步视频 -> 已分析 -> 已生成参考包 -> 已生成草稿 -> 可发布。

**验收标准**

- 首页能看到核心生产指标。
- 用户能判断当前瓶颈在选题、生成、检测还是改写。
- 指标只统计本地工作区数据，不上传用户内容。

## 5. 推荐里程碑

### M1：可用闭环强化

目标：让单条视频可以稳定转成可发布草稿。

- 完成产品命名与主流程引导。
- 增强原文清洗质量提示。
- 完善短片小说结构字段和证据片段。
- 完善 InkOS 失败处理和生成记录。

### M2：质量闭环强化

目标：让用户相信系统能判断“参考结构，但不照抄”。

- 升级相似度、桥段和节奏风险检测。
- 增加风险分级和下一步动作。
- 增加改写前后对比。
- 支持风险段落局部改写。

### M3：规模化生产

目标：让内容工作室能批量选题、批量生产、批量管理。

- 候选素材池和批量分析。
- 项目库排序、看板和批量操作。
- 模板资产库和风格包质量指标。
- 首页生产数据看板。

## 6. 成功指标

### 使用指标

- 单条视频从分析到生成草稿的完成率。
- 从视频报告进入创作工坊的转化率。
- 每周生成创作项目数。
- 每个源视频衍生草稿数。

### 质量指标

- 质量门禁通过率。
- 高风险草稿占比。
- 平均文本重合率。
- 平均改写次数。
- 用户保留最终草稿比例。

### 效率指标

- 单条视频完成清洗、拆解、生成和检测的平均耗时。
- 从 `blocked` 到 `publishable` 的平均轮次。
- 模板复用次数。

## 7. 风险与边界

### 7.1 合规风险

产品不应鼓励复制原文、复用具体人物设定、照搬桥段或规避版权。对外表达建议使用：

- 结构拆解
- 创作转化
- 原创改写
- 风险检测
- 故事创作工坊

避免把“仿写”“洗稿”“规避检测”作为用户侧核心卖点。

### 7.2 质量风险

字幕质量、翻译质量和 LLM 输出质量会影响后续链路。系统需要保留人工编辑入口、置信提示和版本回滚。

### 7.3 成本风险

批量分析、多轮改写和语义检测会增加 LLM 成本。需要任务队列、缓存、手动确认和成本提示。

## 8. 下一步建议

优先做 M1 中的四件事：

1. 把用户侧“仿写工厂”改为“创作转化工坊”，降低合规和产品心智风险。
2. 在首页补强主流程引导，让新用户知道下一步。
3. 增强故事工坊的清洗质量提示和结构证据。
4. 完善 InkOS 生成记录、失败提示和重试。

这四项完成后，项目会更像一个完整产品，而不是一组功能集合。
## 9. 未完成目标 Backlog（2026-06-21 更新）

> 说明：以下内容记录当前仍未完全完成、仍需继续打磨的目标。前面章节中的部分能力已经有基础版本，例如故事工坊、创作转化、项目库、质量门禁、候选素材池和导出交付包；本章节只列出距离“短片小说/故事内容生产工具”完整目标仍有差距的部分。

### 9.1 P0：真实样本闭环验收

**当前状态**

- 当前功能链路已经能覆盖：视频分析、文案清洗、结构拆解、创作参考包、草稿保存、风险检测、改写、项目库和导出。
- 但还缺少用真实 YouTube 故事视频跑完整流程的验收记录，无法证明不同字幕质量、不同题材、不同视频长度下都能稳定产出可审稿内容。

**未完成目标**

- 选取 5-10 条真实短片小说/故事类 YouTube 视频作为验收样本。
- 每条样本完整跑通：同步/导入 -> 分析报告 -> 文案清洗 -> 结构拆解 -> 创作转化 -> 草稿生成 -> 风险检测 -> 改写 -> 导出。
- 记录每条样本的失败点、人工介入点、质量门禁结果和最终导出质量。

**验收标准**

- 至少 5 条样本能完整进入导出阶段。
- 每条样本都有清洗稿、结构拆解、参考包、草稿、风险报告和最终交付包。
- 能明确统计平均耗时、门禁通过率、平均改写次数和高风险失败原因。

**当前状态**

**验收标准**

- 改写后重新检测，语义风险、桥段顺序风险和质量门禁状态能同步变化。
